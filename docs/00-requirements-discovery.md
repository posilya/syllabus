# Сбор требований к проекту

## 1. Общая информация

**Название проекта:** сервис создания и просмотра расписания для учебных заведений.

**Цель документа:** зафиксировать ответы автора проекта, первичные требования, ограничения, открытые вопросы и решения, которые будут использоваться для составления технического задания.
## 2. Общая идея проекта

### 2.1. Тип учебного заведения

Сервис создаётся для расписания колледжа УКСАП.

При этом систему планируется проектировать так, чтобы в будущем её можно было внедрять и в других учебных заведениях:

- колледжах;
- университетах;
- в перспективе школах.

### 2.2. Основная проблема

На текущий момент расписание составляется в Excel-таблицах и вручную публикуется на сайте колледжа.

Сервис должен решить следующие проблемы:

- упростить составление расписания;
- упростить просмотр расписания студентами и преподавателями;
- снизить количество ошибок при составлении расписания;
- обеспечить более быстрый доступ к актуальному расписанию;
- в будущем уведомлять пользователей об изменениях в расписании.

### 2.3. Почему текущее решение неудобно

Текущее решение неудобно по следующим причинам:

- расписание ведётся в Excel;
- расписание публикуется вручную;
- при составлении расписания часто возникают ошибки;
- пользователи не всегда видят актуальные изменения;
- изменения расписания не сопровождаются автоматическими уведомлениями.

### 2.4. Основные пользователи системы

Основные пользователи системы:

- студенты;
- преподаватели;
- администраторы;
- составители расписания.
## 3. Пользователи и роли

### 3.1. Роли в системе

В первой версии MVP предусмотрены следующие роли:

| Роль                               | Описание                                                                                                    |
| ---------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| Гость                              | Неавторизованный пользователь, который может просматривать опубликованное расписание                        |
| Пользователь / Редактор расписания | Авторизованный пользователь, который может редактировать расписание и справочники                           |
| Администратор                      | Пользователь с расширенными правами, который может управлять расписанием, справочниками и учётными записями |

### 3.2. Права ролей

| Действие                             | Гость | Редактор расписания | Администратор |
| ------------------------------------ | ----: | ------------------: | ------------: |
| Просмотр расписания                  |    Да |                  Да |            Да |
| Редактирование расписания            |   Нет |                  Да |            Да |
| Редактирование списка групп          |   Нет |                  Да |            Да |
| Редактирование списка преподавателей |   Нет |                  Да |            Да |
| Редактирование списка дисциплин      |   Нет |                  Да |            Да |
| Редактирование списка аудиторий      |   Нет |                  Да |            Да |
| Управление пользователями            |   Нет |                 Нет |            Да |
| Создание пользователей               |   Нет |                 Нет |            Да |
| Сброс паролей                        |   Нет |                 Нет |            Да |
| Удаление учётных записей             |   Нет |                 Нет |            Да |

### 3.3. Регистрация пользователей

Свободная регистрация пользователей в системе не предусмотрена.

Пользователи создаются администратором.

### 3.4. Доступ к расписанию

Расписание должно быть доступно без авторизации.

В MVP основная часть пользователей системы будет пользоваться расписанием как гости.
## 4. Данные расписания

### 4.1. Данные занятия

Карточка занятия должна содержать следующие данные:

| Поле                   | Описание                            | Обязательность                                               |
| ---------------------- | ----------------------------------- | ------------------------------------------------------------ |
| Дата занятия           | Дата проведения занятия             | Обязательно                                                  |
| Время начала           | Время начала занятия                | Обязательно, если заполнено время окончания                  |
| Время окончания        | Время окончания занятия             | Обязательно, если заполнено время начала                     |
| Группа                 | Учебная группа                      | Должно быть заполнено хотя бы одно из основных полей занятия |
| Преподаватель          | Преподаватель занятия               | Должно быть заполнено хотя бы одно из основных полей занятия |
| Аудитория              | Аудитория / кабинет                 | Должно быть заполнено хотя бы одно из основных полей занятия |
| Дисциплина             | Учебная дисциплина                  | Должно быть заполнено хотя бы одно из основных полей занятия |
| Описание / комментарий | Дополнительная информация к занятию | Необязательно                                                |

### 4.2. Комментарий к занятию

Комментарий к занятию не является обязательным.

В комментарии может указываться дополнительная информация, например:

- ссылка на онлайн-комнату;
- пояснение к занятию;
- информация о замене;
- другая служебная информация.

### 4.3. Тип занятия

Тип занятия считается полезным, но в MVP может быть заменён полем комментария.

В будущем может быть добавлен отдельный справочник типов занятий:

- лекция;
- практика;
- лабораторная работа;
- экзамен;
- консультация;
- другое.

### 4.4. Чётные и нечётные недели

Поддержка чётных и нечётных недель полезна, но не входит в MVP.

После MVP можно добавить возможность указывать периодичность занятия:

- каждую неделю;
- по чётным неделям;
- по нечётным неделям.

### 4.5. Учебные периоды

Поддержка семестров и учебных периодов может быть реализована после MVP.

### 4.6. Корпуса и здания

В MVP корпус может указываться в названии аудитории.

Примеры:

- `П-407`, где `П` означает пристрой;
- `8-906`, где `8` означает номер корпуса.

Отдельный справочник корпусов может быть добавлен после MVP.

### 4.7. Вместимость и оборудование аудиторий

В MVP вместимость и оборудование аудиторий не учитываются.

После MVP может быть добавлена информация:

- о вместимости аудитории;
- наличии проектора;
- наличии компьютеров;
- наличии лабораторного оборудования;
- других характеристиках аудитории.

### 4.8. Автоматическое повторение занятий

Автоматическое повторение занятий в MVP не требуется.

В MVP может быть реализовано копирование расписания:

- на день;
- на неделю;
- на месяц;
- для группы;
- для курса;
- для всего учебного заведения.

## 5. Правила и ограничения

### 5.1. Конфликты расписания

В MVP конфликты расписания не запрещают сохранение занятия.

Система должна выводить предупреждение в редакторе, если обнаружены следующие конфликты:

| Конфликт                                                | Поведение системы      |
| ------------------------------------------------------- | ---------------------- |
| У одной группы два занятия в одно время                 | Вывести предупреждение |
| Один преподаватель назначен на два занятия одновременно | Вывести предупреждение |
| Одна аудитория занята двумя занятиями одновременно      | Вывести предупреждение |

### 5.2. Создание и редактирование занятий в прошлом

В MVP разрешено создавать и редактировать занятия в прошлом.

В будущем это действие может быть ограничено правами доступа.

### 5.3. Удаление занятий

Занятия можно удалять.

Перед удалением система должна запрашивать подтверждение действия.

Отмена занятий как отдельный статус может быть добавлена после MVP.

### 5.4. Поведение при конфликте

При обнаружении конфликта расписания система должна выводить предупреждение в редакторе. 

В MVP предупреждения не запрещают сохранение занятия.

## 6. Просмотр расписания

### 6.1. Параметры просмотра

Пользователь должен иметь возможность просматривать расписание:

- по группе;
- по преподавателю;
- по аудитории / кабинету;
- за день;
- за неделю;
- в перспективе за месяц.

### 6.2. Представление расписания

Возможные варианты отображения расписания:

| Представление                       | Статус                        |
| ----------------------------------- | ----------------------------- |
| Список                              | Требуется                     |
| Таблица за неделю по группам        | Требуется                     |
| Таблица за неделю по преподавателям | Требуется                     |
| Сетка расписания                    | Полезно                       |
| Календарь                           | Не требуется, если будет iCal |

### 6.3. Расписание на сегодня

Система должна позволять просматривать расписание на текущий день.

### 6.4. Расписание на неделю

Система должна позволять просматривать расписание на неделю.

### 6.5. Расписание на месяц

Просмотр расписания на месяц возможен после MVP.
### 6.6. Поиск

Система должна поддерживать поиск:

- по группе;
- по преподавателю;
- по аудитории.
### 6.7. Отображение изменений

Выделение изменений в расписании в MVP не требуется.

### 6.8. Отменённые занятия

Отображение отменённых занятий может быть полезно в будущих версиях.

### 6.9. Прошедшие занятия

Система должна позволять просматривать прошедшие занятия.

## 7. Администрирование

### 7.1. Разделы админки

В админке должны быть следующие разделы:

- редактирование расписания;
- редактирование справочника групп;
- редактирование справочника преподавателей;
- редактирование справочника дисциплин;
- редактирование справочника аудиторий;
- редактирование пользователей, только для администратора.

### 7.2. Управление справочниками

В MVP управление справочниками доступно пользователям с правами редактирования.

После MVP доступ к справочникам может быть ограничен более детальной системой прав.

Справочники MVP:

- группы;
- преподаватели;
- аудитории;
- дисциплины.

### 7.3. Импорт расписания

Импорт расписания из файла в MVP не требуется.

После MVP может быть реализован API для массового добавления и изменения данных.

### 7.4. Экспорт расписания

Экспорт расписания требуется после MVP.

Подробности описаны в разделе "Экспорт и интеграции".

### 7.5. Подтверждение удаления

Перед удалением занятия или записи из справочника система должна запрашивать подтверждение.

### 7.6. История изменений

История изменений нужна, но не входит в MVP.

После MVP система должна хранить историю изменений расписания.

## 8. Уведомления

### 8.1. Необходимость уведомлений

Уведомления об изменениях расписания нужны после MVP. В MVP уведомления не требуются.

## 9. Экспорт и интеграции

### 9.1. Экспорт в PDF

Экспорт расписания в PDF нужен, но не входит в MVP.

PDF-файл должен поддерживать следующие варианты расписания:

- расписание группы на неделю;
- расписание преподавателя на неделю;
- расписание учебного заведения на неделю;
- расписание курса на неделю.

### 9.2. Экспорт в iCal

Экспорт расписания в iCal нужен, но не входит в MVP.

iCal планируется реализовать в одной из первых версий после выпуска MVP.

Пользователи iCal:

- студенты;
- преподаватели.

### 9.3. Импорт из Excel

Импорт расписания из Excel не требуется.

### 9.4. Интеграции с другими системами

Интеграции с другими системами планируются через API.

### 9.5. Публичная ссылка на расписание

Публичная ссылка на расписание нужна.

Публичная ссылка не должна раскрывать персональные данные студентов.

### 9.6. API для внешних систем

API для внешних систем требуется после MVP.
### 9.7.Какие операции API должны поддерживаться в первую очередь

В первую очередь API должно поддерживать операции, необходимые для реализации MVP:

1. Получение расписания для публичного просмотра.
2. Получение справочников групп, преподавателей, аудиторий и дисциплин.
3. Создание, редактирование и удаление занятий.
4. Управление справочниками.
5. Авторизация пользователей с правами редактирования.
6. Управление пользователями администратором.

Операции экспорта, уведомлений, истории изменений, iCal и внешних интеграций относятся к развитию после MVP.

## 13. Предварительные границы MVP

### 13.1. В MVP входит

В первую версию MVP входит:

- просмотр расписания без авторизации;
- просмотр расписания по группе;
- просмотр расписания по преподавателю;
- просмотр расписания по аудитории;
- просмотр расписания за день;
- просмотр расписания за неделю;
- поиск по группе, преподавателю и аудитории;
- авторизация пользователей с правами редактирования;
- редактирование расписания;
- создание занятий;
- редактирование занятий;
- удаление занятий;
- подтверждение перед удалением;
- управление справочниками групп;
- управление справочниками преподавателей;
- управление справочниками аудиторий;
- управление справочниками дисциплин;
- управление пользователями администратором.

В MVP не входит:

- уведомления;
- экспорт в PDF;
- экспорт в iCal;
- импорт из Excel;
- API для внешних систем;
- история изменений;
- отдельный справочник корпусов;
- проверка вместимости аудиторий;
- проверка оборудования аудиторий;
- проверка рабочего времени преподавателя;
- автоматическое повторение занятий;
- поддержка чётных и нечётных недель;
- поддержка семестров и учебных периодов;
- отображение отменённых занятий;
- отдельный статус "Отменено";
- детальная система прав;
- выделение изменений в расписании.
