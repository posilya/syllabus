# Пользователи, роли и права доступа

## 1. Назначение документа

Документ описывает пользователей системы, роли и права доступа в MVP сервиса создания и просмотра расписания учебного заведения.

Раздел фиксирует, какие действия доступны каждой роли, кому требуется авторизация и какие ограничения действуют в первой версии системы.

## 2. Роли системы

В MVP предусмотрены следующие роли:

| Роль | Описание |
|---|---|
| Гость | Неавторизованный пользователь, который может просматривать расписание без входа в систему |
| Редактор расписания | Авторизованный пользователь, который может создавать, изменять и удалять занятия, а также управлять справочниками |
| Администратор | Авторизованный пользователь с расширенными правами, который может управлять расписанием, справочниками и учётными записями пользователей |

В MVP студенты и преподаватели просматривают расписание без авторизации и относятся к роли гостя.

## 3. Описание ролей

### 3.1. Гость

Гость — неавторизованный пользователь системы.

Гость может просматривать расписание без входа в систему.

К гостям относятся:

- студенты;
- преподаватели;
- другие пользователи, которым нужен только просмотр расписания.

Гость не может создавать, изменять или удалять данные в системе.

### 3.2. Редактор расписания

Редактор расписания — авторизованный пользователь, который отвечает за работу с расписанием.

Редактор расписания может:

- создавать занятия;
- редактировать занятия;
- удалять занятия;
- просматривать расписание;
- управлять справочниками групп;
- управлять справочниками преподавателей;
- управлять справочниками аудиторий;
- управлять справочниками дисциплин.

Редактор расписания не может управлять учётными записями пользователей.

### 3.3. Администратор

Администратор — авторизованный пользователь с расширенными правами.

Администратор может:

- просматривать расписание;
- создавать, редактировать и удалять занятия;
- управлять справочниками;
- создавать пользователей;
- редактировать пользователей;
- сбрасывать пароли пользователей;
- удалять учётные записи пользователей.

| Действие                               | Гость | Редактор расписания | Администратор |
| -------------------------------------- | :---: | :-----------------: | :-----------: |

| Поиск расписания                       |  Да   |         Да          |      Да       |
| Создание занятия                       |  Нет  |         Да          |      Да       |
| Редактирование занятия                 |  Нет  |         Да          |      Да       |
| Удаление занятия                       |  Нет  |         Да          |      Да       |
| Управление справочником групп          |  Нет  |         Да          |      Да       |
| Управление справочником преподавателей |  Нет  |         Да          |      Да       |
| Управление справочником аудиторий      |  Нет  |         Да          |      Да       |
| Управление справочником дисциплин      |  Нет  |         Да          |      Да       |

## 5. Правила доступа

### 5.1. Доступ к просмотру расписания

Просмотр расписания доступен без авторизации.

Неавторизованный пользователь может просматривать расписание по группе, преподавателю и аудитории.

### 5.2. Доступ к редактированию расписания

Редактирование расписания доступно только авторизованным пользователям с ролью редактора расписания или администратора.

Гость не имеет доступа к функциям создания, редактирования и удаления занятий.

### 5.3. Доступ к управлению справочниками

Управление справочниками доступно редактору расписания и администратору.

В MVP к справочникам относятся:

- группы;
- преподаватели;
- аудитории;
- дисциплины.

### 5.4. Доступ к управлению пользователями

Управление пользователями доступно только администратору.

Редактор расписания не может создавать, редактировать или удалять пользователей.

Гость не имеет доступа к управлению пользователями.

## 6. Авторизация и регистрация

### 6.1. Вход в систему

Авторизованные пользователи входят в систему по логину и паролю.

### 6.2. Регистрация пользователей

Свободная регистрация пользователей в MVP не предусмотрена.

Пользователей создаёт администратор.

### 6.3. Первичный пароль

Первичный пароль пользователю задаёт администратор.

Также возможна генерация первичного пароля системой.

### 6.4. Получение пароля пользователем

Пользователь получает первичный пароль от администратора.

### 6.5. Изменение пароля

Пользователь может самостоятельно изменить свой пароль.

### 6.6. Сброс пароля

В MVP сброс пароля выполняется администратором вручную.

После MVP может быть реализована отправка ссылки для сброса пароля на email пользователя.

## 7. Ограничения MVP

В MVP не предусмотрены:

- свободная регистрация пользователей;
- отдельные личные кабинеты студентов;
- отдельные личные кабинеты преподавателей;
- восстановление пароля через email;
- блокировка пользователей вместо удаления;
- детальная настройка прав доступа;
- разделение редакторов по отделениям, корпусам, группам или другим признакам.

## 8. Возможное развитие после MVP

После MVP может быть реализовано:

- восстановление пароля через ссылку на email;
- блокировка пользователя вместо удаления учётной записи;
- более детальная система прав доступа;
- разделение прав редакторов по отделениям, корпусам или группам;
- отдельные роли для студентов и преподавателей, если в системе появятся личные кабинеты.
