# Нефункциональные требования

### 1. Поддерживаемые устройства

NFR-001. Система должна поддерживать работу на desktop-устройствах.

NFR-002. Система должна поддерживать работу на мобильных устройствах.

### 2. Поддерживаемые браузеры

NFR-003. В MVP система должна корректно работать в актуальных версиях современных браузеров.

NFR-004. Список конкретных браузеров для тестирования будет уточнён на этапе реализации.

### 3. Производительность

NFR-005. Для MVP точное время загрузки страниц не определяется.

NFR-006. Нагрузочное тестирование не входит в MVP.

NFR-007. Требования к масштабированию не входят в MVP.

NFR-008. Система должна обеспечивать корректную загрузку страниц в демонстрационном сценарии использования.

### 4. Количество одновременных пользователей

NFR-009. Для MVP точное количество одновременных пользователей не определяется.

NFR-010. Требования к поддержке высокой нагрузки не входят в MVP.

NFR-011. Проверка работы системы под высокой нагрузкой не входит в MVP.

### 5. Логирование

### 5.1. Логирование ошибок системы

NFR-012. В MVP логирование ошибок системы не требуется.

### 5.2. Логирование действий пользователей

NFR-013. В MVP логирование действий редактора расписания и администратора не требуется.

NFR-014. История изменений расписания может быть реализована после MVP.

### 6. Резервное копирование

NFR-015. В MVP резервное копирование базы данных не требуется.

### 7. Требования к паролям

NFR-016. В MVP отдельные требования к сложности паролей пользователей не определены.

NFR-017. Авторизованные пользователи входят в систему по логину и паролю.

### 8. Ограничения MVP

В MVP не входят:

- нагрузочное тестирование;
- требования к масштабированию;
- точное определение количества одновременных пользователей;
- логирование ошибок системы;
- логирование действий редактора расписания и администратора;
- история изменений расписания;
- резервное копирование базы данных;
- отдельные требования к сложности паролей.
