
# Функциональные требования

### 1. Просмотр расписания

FR-001. Система должна позволять просматривать расписание без авторизации.

FR-002. Система должна позволять просматривать расписание по группе.

FR-003. Система должна позволять просматривать расписание по преподавателю.

FR-004. Система должна позволять просматривать расписание по аудитории.

FR-005. Система должна позволять просматривать расписание за текущий день.

FR-006. Система должна позволять просматривать расписание за выбранную неделю.

FR-007. Система должна позволять просматривать прошедшие занятия.

### 2. Поиск расписания

FR-008. Система должна позволять искать расписание по группе.

FR-009. Система должна позволять искать расписание по преподавателю.

FR-010. Система должна позволять искать расписание по аудитории.

FR-011. Система должна поддерживать поиск по части названия.

FR-012. Если объект не найден, система должна отображать сообщение о том, что ничего не найдено.

FR-013. Для пользователей с правами редактирования система должна предлагать создать новый объект, если он не найден.
### 3. Авторизация

FR-014. Система должна позволять авторизованным пользователям входить в систему по логину и паролю.

FR-015. Система должна ограничивать доступ к разделам редактирования для неавторизованных пользователей.

FR-016. Система должна позволять пользователю изменить свой пароль.

FR-017. Система должна позволять администратору сбросить пароль пользователя вручную.
### 4. Управление занятиями

FR-018. Система должна позволять пользователю с правами редактирования создавать занятие.

FR-019. Система должна позволять пользователю с правами редактирования редактировать занятие.

FR-020. Система должна позволять пользователю с правами редактирования удалять занятие.

FR-021. Перед удалением занятия система должна запрашивать подтверждение действия.

FR-022. Система должна позволять создавать и редактировать занятия в прошлом.

FR-023. После сохранения занятия система должна обновлять расписание для просмотра пользователями.

### 5. Проверки и предупреждения

FR-024. Система должна проверять корректность заполнения данных занятия до сохранения.

FR-025. Система должна отображать ошибки заполнения данных занятия до сохранения.

FR-026. Система должна проверять наличие конфликта по группе при создании или редактировании занятия.

FR-027. Система должна проверять наличие конфликта по преподавателю при создании или редактировании занятия.

FR-028. Система должна проверять наличие конфликта по аудитории при создании или редактировании занятия.

FR-029. При обнаружении конфликта система должна отображать предупреждение пользователю.

FR-030. В MVP предупреждение о конфликте не должно блокировать сохранение занятия.
### 6. Управление справочниками

FR-031. Система должна позволять пользователю с правами редактирования просматривать справочник групп.

FR-032. Система должна позволять пользователю с правами редактирования создавать, редактировать и удалять группы.

FR-033. Система должна позволять пользователю с правами редактирования просматривать справочник преподавателей.

FR-034. Система должна позволять пользователю с правами редактирования создавать, редактировать и удалять преподавателей.

FR-035. Система должна позволять пользователю с правами редактирования просматривать справочник аудиторий.

FR-036. Система должна позволять пользователю с правами редактирования создавать, редактировать и удалять аудитории.

FR-037. Система должна позволять пользователю с правами редактирования просматривать справочник дисциплин.

FR-038. Система должна позволять пользователю с правами редактирования создавать, редактировать и удалять дисциплины.

FR-039. Перед удалением записи из справочника система должна запрашивать подтверждение действия.
### 7. Управление пользователями

FR-040. Система должна позволять администратору просматривать список пользователей.

FR-041. Система должна позволять администратору создавать пользователя.

FR-042. Система должна позволять администратору редактировать данные пользователя.

FR-043. Система должна позволять администратору удалять пользователя.

FR-044. Система должна позволять администратору сбрасывать пароль пользователя.
