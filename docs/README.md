# Документация проекта

## Сервис создания и просмотра расписания для учебных заведений

Документация содержит требования, пользовательские сценарии, описание интерфейса, модели данных, API и диаграммы проекта.

## 1. Сбор и анализ требований

Файл содержит вопросы к заказчику, ответы, открытые вопросы, принятые решения и предварительные границы MVP.

---

## 2. Общее описание проекта и бизнес-процессы

Файл содержит назначение системы, цели проекта, описание предметной области, границы MVP, ограничения, допущения, а также BPMN-диаграммы текущего и целевого процессов.

---

## 3. Бизнес-требования

Файл содержит бизнес-цели, бизнес-требования и ожидаемый результат внедрения системы.

---

## 4. Пользователи, роли и права доступа

Файл содержит описание ролей пользователей, правила доступа и матрицу доступа.

## 5. Функциональные требования

Файл содержит требования к функциям системы.

## 6. Нефункциональные требования

Файл содержит требования к качеству работы системы.

## 7. User Story и Use Case

Файл содержит пользовательские истории, сценарии взаимодействия пользователей с системой и Use Case Diagram.

## 8. Требования к интерфейсу

## 9. Требования к данным и ER-модель
